数据结构和算法相辅相成，它们共同解决的是“快”和“省”的问题，即如何让代码运行得更快，如何让代码更节省存储空间。[课程代码](https://github.com/wangzheng0822/algo)

为什么不直接执行代码去看代码运行的时间和占用内存空间呢？

1. 代码运行时间受环境（配置高低，不同机器）影响
2. 代码运行与数据规模大小有关

所以采用大O分析法，不受上述两个限制。根据该分析法，一般算法可以被归为以下其中一种复杂度：

$$O(1)常量阶，O(\log n)对数阶，O(n)线性阶，O(n\log n)线性对数阶，O(n^{k})次幂阶\\O(a^{n})指数阶, O(n!)阶乘阶$$

其中第一行的均为多项式量级，第二行的则为非多项式量级，通常把时间复杂度为非多项式量级的算法问题叫做NP（Non-Deterministic Polynomial）问题。

- 线性表：数组，链表，队列，栈等
- 非线性表：二叉树、堆、图等

# 数组 Array

---

简单来讲，数组是用线性、连续的一块内存空间来存储相同类型的数据，一般来讲在定义数组时，需要定义数组的大小，即从内存中划出相应大小的一块连续空间用于存放该数组。数组的第一个特点，如上面提到的，**它属于**[线性表的一种]()，每个线性表上的数据最多只有前后两个方向。第二个特点是，连续的内存空间和相同类型的数据。我们常提到的数组具有随机访问的特性便和这有关，即数组适合查找；但是有利就有弊，这个特性使得数组在插入和删除元素时比较麻烦。考虑数组 $$arr = [a, b, c, d, e]$$，如果要把元素 $$x$$ 插入到下标 2 处， 则需要把 $$a[2]$$ 及后面的元素逐个往后移一位；如果不考虑保留原来数组元素相对顺序，则可以直接把 $$a[2]$$ 直接扔到数组末尾，然后把 $$x$$ 塞进去即可。

Q: 问数组和链表的区别？
A: 链表适合插入、删除操作，时间复杂度均为$$O(1)$$；~~数组适合查找操作，查找时间复杂度为O(1)~~。准确来讲，即便是有序的数组，用二分法查找元素，时间复杂度也是 $$O(\log n)$$。正确的表述应为：数组支持随机访问，根据下标随机访问查找元素的时间复杂度为 $$O(1)$$。

针对数组类型，很多语言都提供了容器类，如Java中的ArrayList和C++ STL中的Vector。

# 链表 Linked List

---

在学习链表之前，我们先来讨论一个经典的链表应用场景，那就是LRU缓存淘汰算法。常见的缓存类别有CPU缓存、数据库缓存和浏览器缓存等，缓存是一种提高数据读取性能的技术。缓存的大小有限，当缓存被用满时，如何判断应该清除哪些数据，保留哪些数据呢？常见的三种策略是：FIFO(First In First Out)、LFU(Least Frequently Used)和LRU(Least Recently Used)。

链表相对数组来讲是一种稍微更复杂的数据结构，但是它们两个都是非常基础、极其常见的数据结构，所以也经常被拿来一起比较。还记得定义一个数组时，需要确定它的大小，然后申请一块连续的内存空间，这样的方式对内存的要求比较高，有时候如果内存中没有符合条件的空间，即连续的、足够大的存储空间，那么就会申请失败。然而链表的情况恰恰相反，它并不需要一块连续的内存空间，而是通过指针将一系列零散的内存块串联起来使用，这里指的是内存块的地址不连续。由于链表内存空间不连续的特点，所以链表插入和删除操作都是 $$O(1)$$ 时间复杂度；与之对应的是查找的时间复杂度为 $$O(n)$$。

链表又分为单向链表、循环链表和双向链表。虽然对于同样的数据，双向链表要比单向链表占用更多的内存空间——前置指针prev，但是在实际应用中，双向链表往往能更简洁高效地完成插入和删除操作。考虑以下两种删除情景：

- 删除链表中值等于某给定值的结点
- 删除某给定指针指向的结点

在第一种情况中，无论是单向链表还是双向链表，都需要从head开始查找，直到找到满足要求的结点，然后进行 $$O(1)$$ 的删除，总的时间复杂度均为 $$O(n)$$ 。而在第二种情况中，由于需要把被删除结点的前段和后段连接起来，所以需要知道被删除结点的前指针和后指针，这步操作在双向链表中是 $$O(1)$$ ，因为prev和next指针都可以直接获取；然而在单向链表中，则需要进行 $$O(n)$$ 的查找。对于插入操作，我们也有类似上述的结论。

> 在Java的LinkedHashMap容器的实现原理中，就用到了双向链表这种数据结构。

# 栈 Stack

---

关于栈，有一个生活化的例子，好比一摞叠在一起的盘子，我们放盘子的时候是在顶端从下往上放，取的时候也是在顶端，从上往下拿。所谓栈结构，即为LIFO。从操作特性来看，栈是一种操作受限的线性表，只允许在一端进行插入和删除。从功能上来讲，数组和链表确实能代替栈，但是，**特定的数据结构是对特定场景的抽象**——当某个数据集合只涉及在一端插入和删除数据，并且满足后进先出的特性，此时栈为首选的数据结构。以下是用数组对栈的一种实现，用的语言是Java。

```java
public class ArrayStack{
  private String[] items;  // 数组
  private int count;       // 栈中元素个数，items[count-1]即为栈顶
  private int n;           // 栈的大小

  // 初始化数组，申请一个大小为n的数组空间
  public ArrayStack(int n){
    this.items = new String[n];
    this.n = n;
    this.count = 0;
  }

  // 入栈操作
  public boolean push(String item){
    // 若数组空间不够，直接返回false，入栈失败
    if (count == n) return false;
    // 将item放到下标为count的位置，并更新count
    items[count] = item;
    ++count;
    return true;
  }

  // 出栈操作
  public String pop(){
    // 若栈空，直接返回null
    if (count == 0) return null;
    // 返回下标为count-1的元素，并更新count
    String tmp = items[count - 1];
    --count;
    return tmp;
  }
}
```

## 栈在函数调用中的应用

比较经典的一个应用场景是函数调用栈，操作系统给每个线程分配了一块独立的内存空间，这块内存被组织成“栈”这种结构, 用来存储函数调用时的临时变量。每进入一个函数，就会将临时变量作为一个栈帧入栈，当被调用函数执行完成，返回之后，将这个函数对应的栈帧出栈。

```c
int main() {
   int a = 1; 
   int ret = 0;
   int res = 0;
   ret = add(3, 5);
   res = a + ret;
   printf("%d", res);
   return 0;
}

int add(int x, int y) {
   int sum = 0;
   sum = x + y;
   return sum;
}
```

## 栈在表达式求值中的应用

再来看栈的另一个常见的应用场景，编译器如何利用栈来实现表达式求值。为了方便解释，我将算术表达式简化为只包含加减乘除四则运算，比如：34+13*9+44-12/3。对于这个四则运算，我们人脑可以很快求解出答案，但是对于计算机来说，理解这个表达式本身就是个挺难的事儿。如果换作你，让你来实现这样一个表达式求值的功能，你会怎么做呢？实际上，编译器就是通过两个栈来实现的。其中一个保存操作数的栈，另一个是保存运算符的栈。我们从左向右遍历表达式，当遇到数字，我们就直接压入操作数栈；当遇到运算符，就与运算符栈的栈顶元素进行比较。如果比运算符栈顶元素的优先级高，就将当前运算符压入栈；如果比运算符栈顶元素的优先级低或者相同，从运算符栈中取栈顶运算符，从操作数栈的栈顶取 2 个操作数，然后进行计算，再把计算完的结果压入操作数栈，继续比较。

## 栈在括号匹配中的应用

除了用栈来实现表达式求值，我们还可以借助栈来检查表达式中的括号是否匹配。我们同样简化一下背景。我们假设表达式中只包含三种括号，圆括号 ()、方括号[]和花括号{}，并且它们可以任意嵌套。比如，${[]()[{}]}$ 或 $[{()}([])]$ 等都为合法格式，而 ${[}()]$ 或 [({)] 为不合法的格式。那我现在给你一个包含三种括号的表达式字符串，如何检查它是否合法呢？这里也可以用栈来解决。

我们用栈来保存未匹配的左括号，从左到右依次扫描字符串。当扫描到左括号时，则将其压入栈中；当扫描到右括号时，从栈顶取出一个左括号。如果能够匹配，比如“(”跟“)“匹配，“[”跟“]”匹配，“{”跟“}”匹配，则继续扫描剩下的字符串。如果扫描的过程中，遇到不能配对的右括号，或者栈中没有数据，则说明为非法格式。当所有的括号都扫描完成之后，如果栈为空，则说明字符串为合法格式；否则，说明有未匹配的左括号，为非法格式。

# 队列 Queue

---

CPU处理任务的速度与线程个数不是严格线性正相关；相反，过多的线程数反而会导致CPU频繁切换，处理性能下降。

队列和栈一样，也是一种操作受限的线性表结构。队列支持FIFO，在队尾插入元素，在队头删除元素。队列也可以用数组或链表来实现。下面是用数组实现队列的Java代码。创建时申请一块预设大小的内存空间，并且用两个指针keep track of队头和队尾。

```java
// 用数组实现的队列
public class ArrayQueue {
  // 数组：items，数组大小：n
  private String[] items;
  private int n = 0;
  // head表示队头下标，tail表示队尾下标
  private int head = 0;
  private int tail = 0;

  // 申请一个大小为capacity的数组
  public ArrayQueue(int capacity) {
    items = new String[capacity];
    n = capacity;
  }

  // 入队
  public boolean enqueue(String item) {
    // 如果tail == n 表示队列已经满了
    if (tail == n) return false;
    items[tail] = item;
    ++tail;
    return true;
  }

  // 出队
  public String dequeue() {
    // 如果head == tail 表示队列为空
    if (head == tail) return null;
    // 为了让其他语言的同学看的更加明确，把--操作放到单独一行来写了
    String ret = items[head];
    ++head;
    return ret;
  }
}
```

# 排序

---

最经典和最常见的排序算法有以下几种：冒泡排序、插入排序、选择排序、归并排序、快速排序、计数排序、基数排序、桶排序等。学习理解一个排序算法或者比较不同的排序算法，需要知道它们的**最好时间复杂度**（及对应的原始数据情况），最坏时间复杂度（及对应的原始数据情况）和**平均时间复杂度**等。当我们用大O法分析比较排序算法时，只考虑到高阶项；而在实际工作操作中，排序的可能是10个、100个、1000个数据的规模，所以**系数**、**常数项**和**低阶项**不宜完全忽视，而且也可以用此进一步区分高阶相同的算法。最后，对于基于比较的算法，应该把元素比较大小、元素交换位置的操作数也考虑进去。



**排序算法的空间复杂度**： 原地排序（in place），即空间复杂度为$$O(1)$$ 的排序算法。

**排序算法的稳定性**：如果排序完成后，大小相等的元素之间相对位置没有发生改变，则称为稳定的排序算法。
